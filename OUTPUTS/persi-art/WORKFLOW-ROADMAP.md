# Persi — Workflow Roadmap: what happens where

Updated: 2026-06-12. Companion to NEXT-STEPS.md (the WHAT); this is the WHERE and the
prompts. One rule covers almost everything:

> **Art content → Cowork. Running code → Claude Code.**
> Cowork has the asset pipeline, image QA, and GIF review loop. Claude Code runs on your
> machine: it can launch Persi, watch the HUD, use real git.

## Step-by-step: tool, prompt, output

| # | Work | Tool | Why |
|---|------|------|-----|
| 0 | Sanity: rebuild exe / run from source, verify_phase6 | Claude Code | Your current `dist/Biscuit.exe` is the PHASE 5 build — it has NO sprites. Feedback from it covers behavior only, not art. |
| 1 | Integration fixes from your run feedback (anchors, scale, wrong clips, sniff pause) | Claude Code | run-fix-run loop |
| 2 | Behavior tuning (fidget rates, state durations) + frame-timing tuning (`tools/art/pack.py` ms table → rerun) | Claude Code | tune while watching the live dog |
| 3 | Regenerate 19 flagged clips (OPEN-ITEMS.md specs) | External tool (you) → **Cowork** | pipeline ingestion, palette lock, preview-GIF approval loop |
| 4 | Prop polish (bed etc.) | Cowork | art content |
| 5 | Phase 4 hooks: type-along + window-bark | Claude Code | keyboard/window hooks need live testing |
| 6 | Phase 7 polish: multi-monitor, DPI, perf, HUD off | Claude Code | needs real displays |
| 7 | Phase 8 ship: sign, installer, startup, updates | Claude Code | machine-level |
| — | Git commits/pushes | Claude Code or your terminal | NEVER from Cowork: the sandbox mirror mangles line endings and can truncate files |

Feedback routing: "the dog feels/acts wrong" → Claude Code session. "the dog LOOKS wrong /
animation reads badly" → Cowork session (art pipeline lives there). When in doubt, behavior
= Code, pixels = Cowork.

   (removes the embedded-token URL; auth now goes through Git Credential Manager)
3. Clear the old cached credential: Windows key → "Credential Manager" → Windows
   Credentials → remove any `git:https://github.com` entries.
4. `git checkout -b feature/phase-6-art`
5. `git add -A`
6. `git commit -m "Phase 6: art pipeline, 34 sprite clips + props, manifest-driven renderer"`
7. `git push -u origin feature/phase-6-art`
8. When prompted: sign in via browser, OR username `myusernameisarthur` + the NEW token as
   password. Credential Manager stores it securely from then on.

want. After this push, Claude Code can handle all future git operations.
